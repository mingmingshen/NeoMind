# 迁移指南：v0.6 → v0.7

## 概述

v0.7.0 是一个以质量为核心的版本，包含稳定性改进、完整的 UI 设计体系重构和全面的测试覆盖。与 v0.6.12 相比，没有破坏性的 API 变更。

## 新特性

### 后端稳定性

- **错误处理** — 将 1000+ 个热路径中的 `unwrap()` 调用替换为安全的错误传播机制，覆盖 8 个 crate，防止生产环境崩溃
- **API 输入验证** — 所有 POST/PUT 端点在处理前验证参数，无效输入返回 400 错误
- **设置持久化** — 设置保存至 `data/settings.redb`，服务器重启后保留
- **MQTT 取消订阅** — 自定义 MQTT 主题现在可以取消订阅
- **规则引擎恢复** — 兜底错误恢复机制防止调度器崩溃

### UI 设计体系

- **极光背景与毛玻璃效果** — 全局极光渐变背景，导航栏和页脚采用毛玻璃风格
- **OKLCH 色彩系统** — CSS 色彩令牌从 HSL 迁移至 OKLCH，实现感知均匀的色彩过渡
- **设计令牌** — 所有硬编码的 Tailwind 调色板颜色替换为语义化设计令牌（text-success、bg-error-light、text-accent-orange 等）
- **协调强调色** — 基于 OKLCH 的分类强调色（紫色、橙色、青色、玫红），深浅模式一致
- **字体** — Plus Jakarta Sans（拉丁文）+ Noto Sans SC（中日韩）字体
- **图表色板** — 重新设计图表颜色，视觉区分度更高，无障碍访问性更好
- **统一对话框组件** — 集中化的 UnifiedFormDialog，一致的 z-index、Portal 和遮罩处理
- **设计规范** — 全面的 `DESIGN_SPEC.md` 文档，记录所有 UI 模式和约定

### 前端优化

- **骨架屏加载** — 所有页面使用一致的骨架屏替代加载动画
- **分页** — 所有列表标准化为每页 10 条记录
- **Toast 通知** — 所有 `alert()` 对话框替换为样式化的 toast 通知
- **表单验证** — Agent、设备和规则编辑器实时验证输入并显示行内错误消息
- **确认对话框** — 破坏性操作（删除、重置）需要明确确认
- **错误边界** — React Error Boundaries 优雅地捕获页面渲染失败
- **错误消息** — API 失败时显示用户友好的消息，而非原始错误文本
- **空状态** — 所有列表页在为空时显示有用的引导信息
- **29 个对话框迁移** — 所有表单对话框统一迁移至 UnifiedFormDialog
- **暗模式修复** — 对话框边框可见，嵌套对话框 z-index 层叠正确

### 性能

- 消除 API 轮询风暴、N+1 查询和阻塞调用链
- 基于 TTL 的请求去重，防止重复 API 调用
- Dashboard 组件首次加载不再显示错误状态

### 测试

- 为 neomind-agent、neomind-storage、neomind-rules、neomind-messages、neomind-extension-runner 和 neomind-api 添加了全面的单元测试

## 破坏性变更

无。v0.7.0 与 v0.6.x 完全向后兼容。

## 配置变更

- 设置现在持久化存储在 `data/settings.redb` 中（自动迁移）
- 事件触发冷却时间默认值从 5 秒改为 60 秒（可通过设置配置）

## API 变更

### 新增响应

- 所有 POST/PUT 端点在无效输入时返回 `400 Bad Request` 及描述性错误消息

### 未变更

- 所有现有端点保持向后兼容
- 没有移除端点或更改响应格式
