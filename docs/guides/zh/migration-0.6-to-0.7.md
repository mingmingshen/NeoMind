# 迁移指南：v0.6 → v0.7

## 概述

v0.7.0 是一个以质量为核心的版本，包含稳定性改进、UI 优化和全面的测试覆盖。与 v0.6.12 相比，没有破坏性的 API 变更。

## 新特性

### 后端稳定性

- **错误处理** — 将 1000+ 个热路径中的 `unwrap()` 调用替换为安全的错误传播机制，覆盖 8 个 crate，防止生产环境崩溃
- **API 输入验证** — 所有 POST/PUT 端点在处理前验证参数，无效输入返回 400 错误
- **设置持久化** — 设置保存至 `data/settings.redb`，服务器重启后保留
- **MQTT 取消订阅** — 自定义 MQTT 主题现在可以取消订阅
- **规则引擎恢复** — 兜底错误恢复机制防止调度器崩溃

### 前端优化

- **骨架屏加载** — 所有页面使用一致的骨架屏替代加载动画
- **分页** — 所有列表标准化为每页 10 条记录
- **Toast 通知** — 所有 `alert()` 对话框替换为样式化的 toast 通知
- **表单验证** — Agent、设备和规则编辑器实时验证输入并显示行内错误消息
- **确认对话框** — 破坏性操作（删除、重置）需要明确确认
- **错误边界** — React Error Boundaries 优雅地捕获页面渲染失败
- **错误消息** — API 失败时显示用户友好的消息，而非原始错误文本
- **空状态** — 所有列表页在为空时显示有用的引导信息

### 测试

- 为 neomind-agent、neomind-storage、neomind-rules、neomind-messages、neomind-extension-runner 和 neomind-api 添加了全面的单元测试

## 破坏性变更

无。v0.7.0 与 v0.6.x 完全向后兼容。

## 配置变更

- 设置现在持久化存储在 `data/settings.redb` 中（自动迁移）
- 事件触发冷却时间默认值从 5 秒改为 60 秒（可通过设置配置）

## API 变更

### 新增响应

- 所有 POST/PUT 端点在无效输入时返回 `400 Bad Request` 及描述性错误消息

### 未变更

- 所有现有端点保持向后兼容
- 没有移除端点或更改响应格式
