# NeoMind v0.7.0 Release Plan — "Quality Leap"

**Date**: 2026-04-26
**Version**: 0.7.0 (upgrade from 0.6.12)
**Theme**: Stability, Code Quality, User Experience, Documentation
**Scope**: Existing module optimization — no major new features

---

## Overview

v0.7.0 focuses on making NeoMind production-ready through four pillars:
1. **Backend Stability** — Eliminate crash risks, harden APIs
2. **Frontend Polish** — Consistent, professional UI experience
3. **Code Quality** — Zero TODOs, comprehensive tests, clean codebase
4. **Documentation** — Accurate, up-to-date, bilingual docs

## Part 1: Backend Stability Hardening

### 1.1 Hot-Path `unwrap()`/`expect()` Elimination

Replace `unwrap()` and `expect()` calls in hot paths with proper error propagation (`?`) or explicit error handling (`map_err`, `match`).

**Scope**: ~1,000+ `unwrap()` calls across Rust source files (excluding tests/examples). We target **hot paths only** — not all unwraps.

**Hot Path Definition**: Any code that runs during request handling, event processing, telemetry ingestion, rule evaluation, message delivery, or agent execution. **Excludes**: CLI arg parsing, test setup, static initializers, `main()` early setup, const/static initialization.

**Target Crates and Modules** (ordered by `unwrap()` count):

| Crate | `unwrap()` Count (src/) | Priority Modules | Typical Issues |
|-------|------------------------|-----------------|---------------|
| `neomind-agent` | 343 | `executor`, `llm_backends/*`, `tools/*` | LLM response parsing, tool execution |
| `neomind-storage` | 303 | `timeseries`, `session` | DB read/write results |
| `neomind-api` | 129 | `handlers/*` | API parameter parsing |
| `neomind-rules` | 93 | `engine`, `dsl` | Condition parsing/evaluation |
| `neomind-devices` | 58 | `mqtt`, `telemetry` | Device mapping, telemetry processing |
| `neomind-core` | 81 | `eventbus`, `extension/isolated/*` | Event dispatch, process management |
| `neomind-messages` | 20 | `channels/*`, `manager`, `delivery_log` | Notification delivery |
| `neomind-extension-runner` | 14 | `main`, `ipc_routing` | IPC state, process communication |

**Not in scope** (explicitly excluded):
- `neomind-extension-sdk`: 76 unwraps, but SDK is consumed by third-party extensions — changes may break compatibility
- `neomind-cli`: 5 unwraps, low-risk CLI tool

**Strategy**:
- **Phase 1a (Critical)**: Focus on `neomind-agent` executor + tools, `neomind-storage` timeseries, `neomind-core` eventbus — highest crash risk
- **Phase 1b (High)**: `neomind-api` handlers, `neomind-rules` engine, `neomind-devices` MQTT
- **Phase 1c (Standard)**: Remaining modules
- Hot paths: Replace with `?` propagation or `map_err` → custom error types
- Init/startup code: Allow reasonable `unwrap`/`expect` with clear comments
- Test code: Allow but prefer `assert!`/`assert_eq!` where appropriate

### 1.2 API Input Validation

Add parameter validation to all `POST`/`PUT` endpoints:

- **Agent CRUD**: name length, config validation, mode requirements
- **Extension upload**: file size, format, metadata validation
- **Device registration**: device_id format, type validation
- **Rule creation**: condition syntax, action validation
- **MQTT subscriptions**: topic format, QoS range
- **Telemetry queries**: time range limits, metric name validation

Validation content: field types, string lengths, numeric ranges, required fields.

### 1.3 Critical Path Robustness

- **Agent execution**: Timeout mechanism refinement, LLM call retry with backoff
- **Rule engine**: Catch-all for condition evaluation errors, graceful degradation
- **Message delivery**: Exponential backoff retry for failed notifications
- **Extension process**: Crash recovery with proper state cleanup and re-initialization
- **Storage**: Complete cache consistency guarantees (post-DB-commit updates)

## Part 2: Frontend UI Polish

### 2.1 Global UI Standards Enforcement

- **Loading states**: All page-level loading must use skeleton screens. `Loader2` spinner only for inline/button/dialog-level loading.
- **Pagination**: Default page size = 10 across all pages (devices, agents, messages, data explorer, extensions, rules).
- **Notifications**: Replace all `alert()` calls with toast notifications (Sonner or equivalent).
- **Error display**: Consistent error toast with user-friendly messages.

### 2.2 Page-by-Page Improvements

| Page | Improvements |
|------|-------------|
| **Dashboard** | Device detail panel (status/metrics/commands), metric tooltip on chart hover, complete VisualDashboard interactions |
| **Agents** | Complete `/api/agents/{id}/available-resources` endpoint, enhance agent editor UX |
| **Devices** | Device health monitoring panel, device grouping capability |
| **Rules** | Rule test/simulation feature, rule templates |
| **Messages** | Message search/filter, delivery status details |
| **Data Explorer** | Data export, advanced filtering |
| **Settings** | Config validation feedback, environment variable management |
| **LLM** | Backend config dialog completion (create backend implementation) |

### 2.3 Universal UX Improvements

- **Form validation**: Real-time field validation with inline error messages
- **Confirmation dialogs**: Dangerous operations (delete, restart, reset) require confirmation
- **Empty states**: Guided prompts when no devices/agents/rules exist yet
- **Error boundaries**: React Error Boundary components for graceful failure handling

## Success Criteria

1. **Zero panics**: No `unwrap()` in hot paths (request handling, event processing, telemetry ingestion, rule evaluation, message delivery, agent execution) that could crash production
2. **API safety**: All POST/PUT endpoints validate inputs and return 4xx on invalid data
3. **Test coverage**: All 6 core crates meet their target coverage (agent 50%+, storage 60%+, rules 50%+, extension-runner 40%+, messages 40%+, api 30%+)
4. **UI consistency**: All pages follow skeleton loading, pagination=10, toast notification standards; no `alert()` calls remain
5. **Zero TODOs**: All 13 TODO/FIXME items resolved (2 backend + 11 frontend)
6. **Documentation**: API docs at v0.7.0, all guides updated, migration guide available
